# EIB Green-Lending Regional Dashboard

A Next.js dashboard visualising European Investment Bank green-lending data across European
regions - the interactive frontend for the [eib-green-lending](../README.md) pipeline.

**Architecture note:** this is a pure static frontend. It has no backend and no API routes -
everything is read from two static files in `public/data/` (`nuts_lending.geojson` and
`summary.json`), generated by the Python pipeline's `src/export_frontend_data.py`. This app
never imports Python code and never talks to a server; it only reads those two files.

## Stack

Next.js 14 (App Router) · TypeScript (strict) · Tailwind CSS · shadcn/ui · MapLibre GL JS ·
Recharts

## Local development

```bash
cd frontend
npm install
npm run dev
```

Open [localhost:3000](http://localhost:3000). The dashboard reads whatever is currently in
`public/data/` - to refresh it with the latest pipeline output, run the export step from the
repo root first:

```bash
# from the repo root, with the Python venv activated
python src/export_frontend_data.py --mode real   # or --mode sample
```

then reload the page (no rebuild needed in dev).

## Deploying to Vercel

1. Push this repo to GitHub (already done if you're reading this from the repo).
2. In Vercel, "Import Project" → select this repo.
3. **Set Root Directory to `frontend`** - this is the only non-default setting needed. Vercel
   auto-detects Next.js and configures the build/output settings itself.
4. Deploy. Since there's no backend, this builds as a static Next.js app with zero runtime
   server dependencies.

To update the live data after a fresh pipeline run, re-run `export_frontend_data.py`, commit
the updated files in `public/data/`, and push - Vercel redeploys automatically.

## Project structure

```
frontend/
├── app/                 # App Router pages, layout, global styles
├── components/          # Dashboard components + components/ui (shadcn)
├── lib/                 # Shared TS types and formatting helpers
└── public/data/         # nuts_lending.geojson + summary.json (from the Python pipeline)
```
